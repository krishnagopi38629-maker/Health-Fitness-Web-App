\begin{abstract}
The rapid growth of big data has created the need for scalable and fault-tolerant distributed processing systems. Hadoop has emerged as a widely adopted open-source framework that enables reliable storage and parallel computation over large clusters of commodity hardware. This paper presents a detailed study of Hadoop architecture, key daemons, and the end-to-end data processing workflow. The roles of HDFS, YARN, MapReduce, and supporting services are analyzed to demonstrate how Hadoop achieves efficient large-scale data management. The discussion provides a clear understanding of how Hadoop components cooperate to deliver high performance and reliability in financial analytics and other data-intensive applications.
\end{abstract}

\begin{IEEEkeywords}
Hadoop, HDFS, YARN, MapReduce, Big Data, Distributed Computing

\section{Introduction}
Hadoop is an open-source distributed computing framework designed to store and process massive datasets efficiently. It follows a master--slave architecture and uses multiple daemons to coordinate storage, computation, and resource allocation across clusters.

%---------------------------------
\section{Key Daemons of Hadoop}

\subsection{Name Node}
Manages HDFS metadata such as file names, permissions, and block locations. It acts as the master of the storage layer.

\subsection{Data Node}
Stores actual data blocks and performs read/write operations. It periodically sends heartbeat signals to the Name Node.

\subsection{Secondary Name Node}
Creates checkpoints by merging FSImage and EditLog to prevent metadata overflow.

\subsection{Resource Manager}
Global authority for resource allocation in YARN. It schedules applications and manages cluster resources.

\subsection{Node Manager}
Runs on worker nodes and manages containers while monitoring resource usage.

\subsection{Application Master}
Responsible for managing the lifecycle of a specific application and coordinating tasks.

Hadoop architecture consists of four major components.

\subsection{HDFS}
Provides distributed storage with block replication and fault tolerance.

\subsection{YARN}
Handles resource management and job scheduling across the cluster.

\subsection{MapReduce}
Implements parallel data processing using Map and Reduce phases.

\subsection{Hadoop Common}
Contains shared utilities and libraries required by Hadoop modules.

\subsection{Step 1: Data Ingestion}
Data is imported using tools like Flume, Sqoop, Kafka, or direct upload.

\subsection{Step 2: Storage in HDFS}
Files are split into blocks and replicated across Data Nodes.

\subsection{Step 3: Map Phase}
Mapper tasks process input splits and generate intermediate key--value pairs.

\subsection{Step 4: Shuffle and Sort}
Intermediate data is grouped by keys and transferred to reducers.

\subsection{Step 5: Reduce Phase}
Reducers aggregate results to produce final output.

\subsection{Step 6: Output Storage}
Processed data is stored back in HDFS for analytics.

%---------------------------------
\section{Fault Tolerance in Hadoop}

Hadoop is designed to handle failures automatically.

\subsection{Data Replication}
Each block is replicated (default factor 3). If one node fails, data is still available.

\subsection{Heartbeat Mechanism}
Data Nodes send periodic heartbeats to the Name Node. Missing heartbeat indicates failure.

\subsection{Task Re-execution}
Failed MapReduce tasks are automatically rescheduled on healthy nodes.

%---------------------------------
\section{Performance Optimization Techniques}

\subsection{Data Locality}
Processing is moved near the data to reduce network traffic.

\subsection{Parallel Processing}
Multiple tasks run simultaneously across cluster nodes.

\subsection{Speculative Execution}
Slow tasks are duplicated to avoid bottlenecks.

\subsection{Compression}
Reduces storage usage and speeds up data transfer.

\section{Limitations of Hadoop}

\begin{itemize}
\item Not ideal for real-time processing
\item High latency
\item Name Node bottleneck (older versions)
\item Complex cluster setup
\item Inefficient for small files
\end{itemize}

\section{Enhanced Conclusion}
Hadoop has revolutionized large-scale data storage and processing through its distributed and fault-tolerant architecture. Its combination of HDFS, YARN, and MapReduce enables efficient handling of massive datasets across clusters. Despite certain limitations, Hadoop continues to be a foundational technology in big data ecosystems and remains highly relevant in modern data engineering environments.
